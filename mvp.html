Product system (MVP)

The network of interaction is comprehensive of three components. The experience starts from the setup of the calendar and the installation of the modules, then the user scans the product with the bar code reader and proceeds in storing them in the modules while checking the calendar.

COMPOSITION
Each module installed can be set at different temperatures, therefore having more modules is fundamental for a succesful system that can avoid food waste. One advantage of using a CLOD system is that it is completely customizable to fit everyoneâ€™s kitchen. If one module is empty, CLOD switches off to avoid energy consumption

What's inside?
Click on the blue points to discover more
CLOD Calendar
The user interface is programmed in HTML/CSS/JavaScript and is displayed on a touch screen.
COMPONENTS
  • Touchscreen display
Cloud
Thanks to a Raspberry we created a database with EAN codes, food properties and recipes. Through the database, the system can recognize food and sends instructions to the module and the calendar.
COMPONENTS
  • Raspberry
  • CSVdatabase
CLOD Module
Inside the module we can find an Arduino mkr wifi 1010 which controls the sound speakers, the led lights and the proximity sensors and communicates with the scanner and calendar through the wifi connection.
COMPONENTS
  • Arduino mkr wifi 1010
  • Mp3 DF player
  • Sound speaker
  • Addressable led strip
  • Weight sensor
  • Proximity sensor
CLOD Scanner
Inside the scanner there is an Arduino mkr wifi 1010 that is connected to the barcode reader and communicates with the calendar and module through the wifi connection.
COMPONENTS
  • Arduino mkr wifi 1010
  • GM65 barcode scanner
  • Powerbank
